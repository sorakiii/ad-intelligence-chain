# 环境变量与密钥配置说明

部署或本地运行本仓库前，请自行准备以下密钥与配置。**切勿将真实 `.env` 文件提交到 Git。**

## 快速开始

```bash
# 后端
cp backend/app/.env.example backend/app/.env
# 编辑 backend/app/.env，填入真实值

# 前端（本地开发）
cp frontend/.env.example frontend/.env.development
# 按需修改 VITE_API_URL、VITE_EQMJ_HOME
```

Docker 部署时，通过 `docker-compose` 的 `environment` 或挂载 `.env` 注入变量。

---

## 后端环境变量一览

配置文件路径：`backend/app/.env`（或 `backend/app/.env.development` / `.env.production`，由 `FLASK_ENV` 决定）。

### 必填（核心服务无法启动）

| 变量名 | 说明 | 获取方式 |
|--------|------|----------|
| `SQLALCHEMY_DATABASE_URI` | MySQL 连接串 | 自建或云数据库，格式：`mysql+pymysql://user:pass@host/db?charset=utf8mb4` |
| `JWT_SECRET_KEY` | JWT 签名密钥 | 自行生成随机长字符串 |
| `DIFY_API_URL` | Dify API 根地址 | [Dify](https://dify.ai) 自建或云服务控制台 |
| `REDIS_URL` | Redis 连接 | 本地 `redis://localhost:6379/0`；Docker 开发可用 `redis://:foobared@redis:6379/0`（见 `docker-compose.dev.yml`） |

### Dify 相关（按功能配置）

| 变量名 | 必填 | 用途 |
|--------|------|------|
| `DIFY_API_KEY` | 推荐 | 默认 Dify 应用 Key；聊天角色未单独配置时使用 |
| `DIFY_API_KEY_HTML_ZIP` | HTML 功能需要 | HTML 生成（zip 模式） |
| `DIFY_API_KEY_HTML_RAW` | HTML 功能需要 | HTML 生成（raw 模式） |
| `DIFY_API_KEY_VIDEO_SCRIPT` | 视频脚本解析需要 | 将脚本转为结构化 JSON 后创建视频任务 |
| `DIFY_API_KEY_MJ_PROMPT` | MJ 提示词提取需要 | 从文本提取 Midjourney 提示词 |

Dify Key 格式通常为 `app-` 开头，在 Dify 控制台 → 应用 → API 访问中创建。

**数据库中的角色 Key**：表 `roles.dify_api_key` 可为每个 AI 角色配置独立 Key，优先级高于 `DIFY_API_KEY`。

### 文生视频（阿里云 DashScope）

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `DASHSCOPE_API_KEY` | 使用文生视频时必填 | [阿里云百炼 / DashScope](https://dashscope.aliyun.com/) 控制台创建 |

### Midjourney / EQMJ 绘图

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `MJ_API_URL` | 是 | 绘图服务 Base URL，默认占位 `https://api.example.com` |
| `MJ_API_KEY` | 是 | 接口 Authorization |
| `MJ_APP_KEY` | 视服务商要求 | App-Key 请求头 |

详见 `specs/text-to-image.md` 中的鉴权说明（示例已脱敏）。

### 短信（253 平台）

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `SMS_ACCOUNT` | 使用短信登录/注册时必填 | 253 云通讯账号 |
| `SMS_PASSWORD` | 同上 | 253 平台密码 |
| `SMS_SIGN_NAME` | 同上 | 短信签名，如 `【您的签名】` |

实现见 `backend/app/services/sms.py`。

### 对象存储（华为云 OBS）

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `OBS_ACCESS_KEY` | 上传/预览文件时必填 | 华为云 IAM 访问密钥 |
| `OBS_SECRET_KEY` | 同上 | 华为云 Secret |
| `OBS_ENDPOINT` | 同上 | 如 `https://obs.cn-north-4.myhuaweicloud.com` |
| `OBS_BUCKET` | 同上 | 桶名称 |

### 其它

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `DOMAIN` | 否 | 后端对外域名，默认 `http://localhost:5002` |
| `FLASK_ENV` | 否 | `development` / `production` / `testing` |
| `UPLOAD_FOLDER` | 否 | 本地上传目录 |

## 前端环境变量

文件：`frontend/.env.development`（开发）、`frontend/.env.production`（构建生产包）。

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `VITE_API_URL` | 后端 API 地址；生产构建可为空字符串（同源） | `http://localhost:5002` |
| `VITE_EQMJ_HOME` | 「跳转 EQMJ」外链首页 | `http://localhost:3005/home` |

复制 `frontend/.env.example` 后修改即可。`frontend/.env.development` 与 `.env.production` 已加入 `.gitignore`，请勿提交含生产域名的文件。

## 本地最小可运行配置示例

仅验证前后端启动（不含短信、OBS、AI 能力）：

**backend/app/.env**

```env
FLASK_ENV=development
SQLALCHEMY_DATABASE_URI=mysql+pymysql://root:password@localhost/ai_ad?charset=utf8mb4
JWT_SECRET_KEY=change-me-in-production
DIFY_API_URL=http://your-dify-host/v1
REDIS_URL=redis://localhost:6379/0
DOMAIN=http://localhost:5002
```

**frontend/.env.development**

```env
VITE_API_URL=http://localhost:5002
VITE_EQMJ_HOME=http://localhost:3005/home
```

完整 AI、上传、短信能力需补全上文各分组变量。

## 安全提醒

1. 若本仓库曾包含真实密钥，开源或推送前请在各云平台 **轮换/作废** 旧密钥。
2. 勿将 `backend/app/.env`、`frontend/.env.development`、`backend/test.rest` 提交到公开仓库。
3. 生产环境使用强随机 `JWT_SECRET_KEY`，并与开发环境区分。
4. `docker-compose.dev.yml` 中 Redis 密码 `foobared` 仅用于本地 Docker，勿用于生产。

更多部署步骤见 [README-部署说明.md](../README-部署说明.md)、[backend/README.md](../backend/README.md)。
