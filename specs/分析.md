## 功能说明
1、每天的总体使用量，这个应该看token吧？这样总量token还能算费用
2、每个人每天使用量
3、每天每个板块的使用人数，使用量
4、每天每个人都是用了哪些板块

相关功能用到的表就 4 个，`users`、`chat_sessions`、`chat_messages`、`roles`。
具体定义看下文。

## 权限控制
目前的用户表过于单薄
```sql
CREATE TABLE `users` (
  `id` int NOT NULL AUTO_INCREMENT,
  `phone` varchar(11) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci NOT NULL,
  `password_hash` varchar(128) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci DEFAULT NULL,
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  `last_login` datetime DEFAULT NULL,
  PRIMARY KEY (`id`) USING BTREE,
  UNIQUE KEY `phone` (`phone`) USING BTREE
) ENGINE=InnoDB AUTO_INCREMENT=18 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci ROW_FORMAT=DYNAMIC
```
我觉得可以加个role_id 字段，用于控制用户权限。
譬如 role_id=0 是默认值，是普通用户，不能查看 数据分析模块。
role_id=1 是管理员，可以查看 数据分析模块。

## 数据分析查询
下面的语句可以查询 token 使用量。
```sql
   SELECT 
            24000000 as total_token,
            SUM(COALESCE(jt.total_tokens, 0)) as use_token,
            24000000 - SUM(COALESCE(jt.total_tokens, 0)) AS remain_token
        FROM 
            chat_messages
        JOIN 
            JSON_TABLE(
                chat_messages.api_usage,
                '$'
                COLUMNS(
                    total_tokens INT PATH '$.total_tokens'
                )
            ) AS jt
        WHERE 
            chat_messages.`timestamp` >= '2025-04-11 17:31:54';
```

### 权限要求

所有数据分析API都要求用户具有管理员权限（role_id=1）才能访问。

### 1. 获取总Token使用量

```
GET /api/analytics/total-usage
```

**请求参数**:
- `start_date`: 可选，开始日期，格式为YYYY-MM-DD，默认为当天

**响应示例**:
```json
{
  "data": {
    "total_token": 24000000,
    "used_token": 1500000,
    "remaining_token": 22500000
  },
  "status": "success"
}
```

### 2. 获取用户Token使用量

```
GET /api/analytics/user-usage
```

**请求参数**:
- `start_date`: 可选，开始日期，格式为YYYY-MM-DD，默认为当天

**响应示例**:
```json
{
  "data": [
    {
      "user_id": 1,
      "phone": "13800000001",
      "token_usage": 500000
    },
    {
      "user_id": 2,
      "phone": "13800000002",
```

### 5. 获取每日统计数据

```
GET /api/analytics/daily-stats
```

**请求参数**:
- `start_date`: 可选，开始日期，格式为YYYY-MM-DD，默认为7天前
- `end_date`: 可选，结束日期，格式为YYYY-MM-DD，默认为今天

**响应示例**:
```json
{
  "data": [
    {
      "date": "2023-04-20",
      "unique_users": 5,
      "message_count": 120,
      "token_usage": 150000
    },
    {
      "date": "2023-04-21",
      "unique_users": 8,
      "message_count": 180,
      "token_usage": 200000
    }
  ],
  "status": "success"
}
```

### 6. 设置用户为管理员

```
POST /auth/set-admin
```

**请求体**:
```json
{
  "phone": "13800000002",
  "is_admin": true
}
```

**响应示例**:
```json
{
  "message": "用户 13800000002 设为管理员成功",
  "status": "success"
}
```

### 7. 检查当前用户是否为管理员

```
GET /auth/check-admin
```

**响应示例**:
```json
{
  "data": {
    "is_admin": true
  },
  "status": "success"
}
```
